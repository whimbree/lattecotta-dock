## Recorded deviations from the spec sketch

1. The spec sketches one `BackgroundEnv -> resolved{...}` struct pair.
   Implemented instead as per-predicate functions with per-predicate
   QML bindings, because a single resolved blob recreates the exact
   binding loops the QML structure exists to break:
   viewTypeInQuestion's own comment says it exists "to help with
   binding loops" (background.isGreaterThanItemThickness is an input
   while viewType drives background geometry), and
   panelShadowsActive's background.currentOpacity input feeds back
   from MultiLayered layers whose opacity states read
   forceSolidPanel/forceTransparentPanel - outputs of the same blob.
   Per-predicate bindings keep the QML notify graph identical to
   today's, which is also the minimal live-risk shape. The core is
   still one header and the test drives the predicates composed in
   dependency order, so the "one resolver" intent (single tested
   authority) holds.
2. blurEnabled (main.qml:93) folded into the unit although the spec's
   source list stops at the 126-149 cluster: its only non-config
   inputs are two cluster outputs and its result is a
   panelShadowsActive input - extracting the cluster without it would
   leave one copy of cluster logic behind in QML.
3. normalBusyForTouchingBusyVerticalView (main.qml:151-153) folded
   into isPanelForcedForBusyBackground and the QML property deleted
   (single consumer, verified by grep).

## Findings

- THE DEAD TRANSPARENCY CLAUSE (inherited, Qt5-faithful, pinned not
  fixed): panelShadowsActive's
  `transparencyCheck = blurEnabled || (!blurEnabled && background.currentOpacity > 20)`
  compares a unit-interval opacity (currentOpacity is
  overlayedBackground.backgroundOpacity or solidBackground.opacity,
  both 0..1; backgroundStoredOpacity = panelTransparency/100) against
  20. The clause can never fire; transparencyCheck degenerates to
  blurEnabled. Qt5 f0ad7b23 has the identical line and identical 0..1
  units, and its comment disagrees with both ("greater than 10%").
  Kept bit-faithful per the Qt5-fidelity agreement (a fix changes live
  shadow behavior for the busy-background state); named test
  panelShadows_theDeadTransparencyClause pins the degenerate truth so
  any future "fix" flips a named case deliberately. Flagged for a
  deliberate behavior decision outside this unit.
- Qt5 comparison: all six extracted bodies are token-identical between
  f0ad7b23 and HEAD (only plasmoid -> Plasmoid capitalization). No
  port-era drift to arbitrate.
