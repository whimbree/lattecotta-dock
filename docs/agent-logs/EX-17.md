## Design decisions (made before first commit)

- Core: `plasmoid/plugin/units/tooltiptext.h`, header-only pure
  functions in `Latte::TooltipText`:
  - `composeTitle(display, appName)` - the full Qt5 regex chain
    (counter strip, appName suffix strip, separator strip, counter
    restore, em-dash fallback), first-match-only excision to mirror
    JS non-global `.replace` exactly.
  - `planSubText(SubTextFacts)` -> `SubTextPlan` - every DECISION
    (which lines appear, which i18n variant, the joined arguments).
    The final i18nc catalog lookup is ambient state (locale), which
    the pure-core rules park outside the core, so the plan/compose
    split follows the EX-01 engine/bridge shape: core decides,
    wrapper translates. Deviation from the spec's literal
    `QString composeSubText(TaskEnv)` recorded with this reason; the
    QML-facing wrapper exposes exactly that shape.
  - ActivitiesLine is a std::variant (AvailableOnAllActivities /
    AvailableOn{names} / AlsoAvailableOn{names}) inside
    std::optional - "all-activities line carrying names" and
    "names line without names" are unrepresentable.
- Wrapper: `TooltipTextComposer` QML singleton registered in
  lattetasksplugin (org.kde.latte.private.tasks) - stateless
  Q_INVOKABLEs, i18nc mapping via std::visit. plasmoid/Messages.sh
  already extracts .cpp files into the same translation domain the
  QML i18n used (plasma_applet_org.kde.latte.plasmoid, defined for
  the plugin in plasmoid/CMakeLists.txt line 1), so the moved i18nc
  calls keep their catalog entries.
- QML keeps: value-domain checks (undefined model/display), the
  isGroup source selection, and the LIVE lookups (desktop id -> name,
  activity id -> name); the core receives resolved names.
- Defect found while extracting (fix at origin, own commit):
  ToolTipInstance.qml:474 `desktopNames[virtualDesktops[i] - 1]` is
  the Qt5 numeric-desktop assumption; Plasma 6 libtaskmanager hands
  desktop IDs (UUID strings on wayland) -> NaN index -> undefined
  name -> JS join renders it empty: the line reads "On " / "On ,"
  (measured offscreen, not guessed). ContextMenu.qml:421 already
  uses the id-based desktopIds.indexOf pattern; upstream
  plasma-desktop's ToolTipInstance does the same. latte-dock-ng
  carries the same bug (its ToolTipInstance:494 unchanged from Qt5).
- TaskItem.qml `generateSubText` twin: DEAD CODE - no caller at HEAD
  nor at f0ad7b23 (verified by grep both trees; only the definition
  and ToolTipInstance's own call sites exist). Kept as a thin shell
  per the spec's twin-equivalence contract (both call sites delegate
  to the one composer); its Qt5-only divergences (uncontexted
  i18n("On %1") msgctxt, no empty-activity-name filtering, dead
  Plasma 5 `model.VirtualDesktop` single-desktop role) unify onto the
  live twin's semantics, recorded in the cutover commit body.

## Qt5 fidelity verdicts (f0ad7b23 read directly)

- ToolTipInstance.qml generateTitle (Qt5 lines ~403-445) and
  generateSubText (~447-506): byte-identical to HEAD's bodies (only
  whitespace/position drift). No port drift to reconcile.
- TaskItem.qml:537 generateSubText: identical to HEAD's body modulo
  the mechanical `plasmoid.` -> `Plasmoid.` port rename.
- JS regex semantics carried exactly: \d/\w ASCII in both JS and
  PCRE2-default; JS \s is unicode-aware while PCRE2-default \s is
  ASCII, so the core spells the JS class out ([\p{Z}\t\n\x0B\f\r﻿]).
  JS non-global replace = first-match-only excision in C++.
- Deliberate divergences from Qt5 (each commented at the site):
  - invalid appName regex (e.g. "Notepad++"): Qt5/JS THREW and broke
    the whole title binding; core skips the appName strip instead.
  - undefined display/appName reaching the C++ boundary coerces to
    "" (Qt5 threw); shells keep the Qt5 undefined checks that matter.
  - desktop entry condition moves from raw-id-list length to
    resolved-name-list length (consequence of the id-resolution fix;
    Qt5 could render "On undefined").

## Eliminated invalid states (step-2.5 point b; for the ledger tick)

- "activities line without names" / "all-activities line carrying
  names": ActivitiesLine is a std::variant whose AvailableOnAll
  alternative has no name field and whose name-bearing alternatives
  are only emitted for a non-empty join (proof slots:
  planSubText_activityLines, wrapper i18n mapping via exhaustive
  std::visit).
- "desktop line with zero desktops" ('On '): SubTextPlan.
  desktopNamesJoined is optional and only set non-empty (proof slot:
  planSubText_desktopLineNeverEmpty).
- "unknown Activities role vs empty list" ambiguity:
  std::optional<QList<Activity>>, no sentinel (proof slot:
  planSubText_unknownActivitiesRole).
- "parallel activity id/name lists drifting": Activity{id,name} is
  one value; the wrapper refuses length mismatch and half-null pairs
  at the boundary with qCritical (proof slot:
  wrapper_refusesMalformedFacts).
