## Design (recorded before code)

Four hand-rolled wheel handlers, three behaviors, one core:

| site | axis pick | firing | units |
|---|---|---|---|
| AudioStream.qml onWheel | y if nonzero else -x, honoring wheel.inverted | accumulate, fire every full step, residue carries, reset on reversal | step 120 |
| TaskMouseArea.qml onWheel | dominant axis (strict abs(y)>abs(x), ties -> horizontal) | fire once past strict threshold, no residue | threshold 96 (Qt5 angle>12) |
| EnvironmentActions.qml onWheel | signed extreme: both>=0 -> max(y,x), else min(y,x) (Qt5 quirk, preserved) | fire once past strict threshold | threshold 80 (Qt5 angle>10) |
| RulerMouseArea.qml onWheel | vertical only | fire once past strict threshold | threshold 96 (Qt5 angle>12) |

Threshold unit mapping: Qt5 compared angle = angleDelta/8 against 12
(or 10) in JS floats; angleDelta components are ints, so angle > 12
is exactly angleDelta > 96 (and angle > 10 exactly > 80) - integer
comparison, no float needed.

Core `declarativeimports/core/units/wheelaccumulator.h` (header-only,
pure, no Qt event types beyond QPoint):

- `enum class WheelAxisPick { VerticalElseNegatedHorizontal,
  DominantAxis, SignedExtreme, VerticalOnly }`
- firing policy is a std::variant, so the invalid combination
  "resetOnReversal in threshold mode" is unrepresentable:
  `struct AccumulateEveryStep { int stepSize; bool resetOnReversal; }`
  / `struct FireOncePastThreshold { int threshold; }`
- `class WheelAccumulator { int add(QPoint angleDelta, bool inverted
  = false); }` returning the signed fired step count. Only the
  accumulate policy holds state (the residue).
- `static bool verticalIsDominant(QPoint angleDelta)` - the single
  authority for the dominant-axis predicate; TaskMouseArea's
  parallel-scroll read uses it through the wrapper so the axis
  decomposition never exists twice.

DEVIATIONS from the spec's literal interface, reasons:
1. `add` takes (QPoint, bool inverted), not QPoint alone: AudioStream
   honors wheel.inverted (299a241b, plasma-pa semantics). inverted is
   a sign flip on the picked axis delta, not an event type; the other
   three sites pass false (Qt5 ignored inverted there - fidelity).
2. `verticalIsDominant` exposed next to add(): TaskMouseArea computes
   parallelScrolling from the axis choice; returning int-only from
   add() would force the shell to re-derive the axis pick by hand -
   the exact duplication this unit exists to end.
3. The spec's constructor bag {stepSize, axisPriority,
   resetOnReversal} carries no mode and makes resetOnReversal
   representable in threshold mode; the step-2.5 type law wins:
   axis pick enum + firing-policy variant.

Boundary wrapper `declarativeimports/core/wheelstepper.{h,cpp}`,
registered as instantiable `LatteCore.WheelStepper` (per-instance
state: each audio badge owns its own residue - a singleton cannot
serve N badges). Properties: axisPick (Q_ENUM, REQUIRED), stepSize
(0 = unset), resetOnReversal, fireThreshold (-1 = unset).
Exactly one of stepSize/fireThreshold must be configured;
misconfiguration refuses loudly at add() (qCritical, 0 steps) - the
boundary is where bad input can arrive from outside.

Qt5 arbitration: sites 2-4 byte-match Qt5 f0ad7b23 already (verified
by reading both trees); site 1 deliberately deviates per 299a241b
(plasma-pa-matched semantics WIN - deviation recorded in that commit
body and in the spec).

## Findings

- Sites 2-4 are identical to their Qt5 f0ad7b23 bodies (read both);
  site 1 deviates deliberately per 299a241b. No new drift found.
- EnvironmentActions' SignedExtreme pick is a real Qt5 quirk worth
  pinning: mixed signs take the min, so a strong upward y with any
  negative x (y=200, x=-1) picks -1 and fires NOTHING. Preserved
  exactly (Qt5-faithful), pinned by dedicated test rows so nobody
  "fixes" it silently later.
- DominantAxis ties (abs(y) == abs(x)) pick HORIZONTAL (Qt5's strict
  >); to be pinned.
- A zero-delta event ((0,0)) must not reset the accumulate residue:
  the QML sign checks treat 0 as neither direction. To be pinned.

## Eliminated invalid states (draft; finalized at closeout)

- resetOnReversal in threshold mode: unrepresentable (firing policy
  is std::variant; the flag exists only inside AccumulateEveryStep).
- stepSize <= 0 (infinite fire loop) and threshold < 0 (fires on
  zero-delta events): Q_ASSERT preconditions in the core ctor (live
  under QT_FORCE_ASSERTS in the sanitized tests), qCritical refusal
  at the WheelStepper boundary where QML config arrives.
- Unconfigured/doubly-configured wrapper mode (neither or both of
  stepSize/fireThreshold): loud qCritical refusal at add(), returns
  0 steps.
- Residue in threshold mode: no member to hold it (state lives in
  the variant alternative).
