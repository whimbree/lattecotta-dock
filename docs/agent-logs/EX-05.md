  (isModernDockStyle does not exist here); its CompactApplet half caps

  reason in the core header). The bound holds on step-1/fair-share
  paths and is asserted there.

  `adjustedSize = curApplet.isHidden ? 0 : properSize` re-check was
  unreachable (the participation predicate already excludes hidden).

## Type discipline (eliminated invalid states, for the ledger)

- The [availableSpace, sizePerApplet, noOfApplets] positional JS array
  shuttle (res[0]/res[1]/res[2]) -> DistributionBudget named struct;
  misindexing unrepresentable.
- The inMaxAutoFillCalculations bool threaded through every call ->
  enum class FillPass.
- computeStep2's noOfApplets sentinel (-1/0/positive, where the else
  branch never used the count) -> enum class Step2Mode; a "count" that
  means a mode is unrepresentable.
- -1 metric sentinels -> std::optional<double> NormalizedMetrics
  inside the core; the single surviving -1 is the documented Qt5
  fallback inside boundPreferredLength (equivalence argument for
  folded negative prefs written at normalizeMetricsForFill).
- Untouched-vs-assigned no longer encoded by writing-or-not-writing a
  live property mid-loop: ItemAssignment's optionals make untouched
  explicit at the boundary.
- The undefined-until-found mostDemandingApplet JS var -> a plain
  pointer scan confined to one small function; assignment happens at
  a single write point (assignFillLength) that owns the coercion.
- Boundary refusal: wrapper qCriticals and refuses missing keys and
  negative counters; core Q_ASSERTs (live under QT_FORCE_ASSERTS)
  pin min-present-at-bound and the dispatcher's fillApplets guard.
