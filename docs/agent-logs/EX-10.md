## Design (recorded before code)

Core: containment/plugin/units/maskgeometry.h, namespace
Latte::MaskGeometry, header-only, two pure functions mirroring the
QML pair:

- `computeLocalGeometry(LocalGeometryInputs) -> QRectF`: the
  per-edge local-geometry table of updateMaskArea. Panel mode
  returns the full root rect (Qt5: clamps only apply on the
  !behaveAsPlasmaPanel path). Dock mode: length axis from
  latteView.effects.rect, thickness axis = metrics.totals.thickness
  offset by metrics.mask.screenEdge, then the inherited boundary
  clamps (x/y bounded to [0, viewSize], width/height min'd with
  viewSize) - assessed as DOCUMENTED NORMALIZATION, not bandaid:
  effects rect and window size arrive from different signal
  cascades, and Effects::setInputMask explicitly expects and clears
  degenerate warmup rects. Comment carries the assessment.
- `computeInputMask(InputMaskInputs) -> InputMaskDecision` where
  `InputMaskDecision = std::variant<AcceptAllInput,
  AcceptInputWithin, AcceptNoInput>`: thickness selection
  (hidden / floating-gap-input-disabled / normal, each with the
  11f42978 parabolicAnimating alternative), subtractedScreenEdge,
  per-edge rect table (length axis from localGeometry only when not
  animating), same clamps, then the sidebar+hidden override.

Type discipline (eliminated invalid states, for the ledger):
- The effects-protocol sentinel rects - Qt.rect(0,0,-1,-1) "accept
  everywhere" and Qt.rect(-1,-1,1,1) "accept nowhere" - are
  UNREPRESENTABLE in the core: they exist only at the wrapper
  boundary, mapped from the variant alternatives via
  std::visit + static_assert exhaustive dispatch (the AutoSizeStepper
  model). No magic rects inside the math.
- location is Plasma::Types::Location in the core (switch), not an
  int chain; non-edge locations (Floating during startup) are the
  explicit documented Qt5 fall-through (full root rect), not a
  silent else.
- Negative dimensions/thicknesses assert in the core (never-states);
  zero sizes pass through as the documented warmup protocol (a
  loud refusal would spam every startup for a legitimate state -
  recorded deviation from the spec's "degenerate window sizes
  rejected loudly" bullet, reason: effects.cpp's warmup contract
  makes zero-size a protocol state, not a defect; negative is the
  assertable never-state).

Geometry math in qreal throughout; the wrapper converts to QRect via
QRectF::toRect(), which is byte-identical to what the QML
rect -> QRect Q_PROPERTY assignment did (pinned by a fractional
zoomedForItems test case).

Wrapper: containment/plugin/maskgeometrybridge.{h,cpp}, QObject
registered as LatteContainment.MaskGeometryBridge (AutoSizeStepper
precedent). Two invokables mirroring the pair; positional args read
at call time (deliberately NOT bound properties: binding refresh vs
signal-handler ordering is not guaranteed, and a stale-by-one-update
mask is the invisible-dock class). Boundary validation mirrors
AutoSizeStepper: qCritical-and-refuse on negative sizes/thicknesses
and unknown location ints, degrading to the pre-mask behavior
(full-rect local geometry / accept-all input) so a refused call
never bricks input.

Shell cutover: VisibilityManager.qml keeps the gates

inNormalState apply-gate with the hidden-breaks-Dodge comment, debug

## Findings

- Provenance: input half is upstream 11f42978, not port-era (spec
  corrected in place).
- Upstream dead code: localX/localY (updateMaskArea),
  floatingInternalGapAcceptsInput (updateInputGeometry) - dead since
  f0ad7b23, deleted with the cutover bodies.
- The localGeometry qBound clamps and the width/height-only min
  (no zero floor on spans) are Qt5-verbatim and load-bearing for the
  warmup protocol; carried with the effects.cpp citation.
- Deviation from spec test-plan bullet "degenerate window sizes
  rejected loudly": zero-size inputs are the documented warmup
  protocol (effects.cpp clears the mask for them); the core asserts
  only on NEGATIVE dimensions and the wrapper refuses those loudly.
  Recorded in the spec section.

## Pending live checks (for the orchestrator - the spec's mandatory matrix)

All recipes assume the staged throwaway config
(scripts/restart-staged.sh) and the latte-live-verification skill's
traps (glides ~8px steps, never jumps; stop the dock before any
rebuild; first kglobalaccel invoke races).

1. Visibility-mode reveal/hide, per mode (dodge-active,
   dodge-maximized, auto-hide, windows-go-below):
   - Set the mode on the throwaway dock (Behavior page or config
     round-trip), open a real window (e.g. dolphin), drive it against
     the dock edge (move/maximize), verify the dock hides; glide the
     pointer to the screen edge, verify reveal. dumpwins before/after
     each transition: the dock window geometry must be unchanged
     (masking, not resizing) for behaveAsDockWithMask.
2. Input-edge probe (the dead-input class), bottom edge, normal
   state: fakepointer CLICK just inside the input band
   (y = screenHeight - 2, x = dock center) -> dock reacts (task
   hover/press). Click just OUTSIDE the mask length-axis bounds
   (x = localGeometry.x - 10 at the same y, i.e. in the dock
   window's margin but outside effects.rect) -> the click must fall
   through to the desktop (desktop context menu or window below
   receives it), NOT the dock.
3. Auto-hide hidden state: after the dock hides, the input band is
   metrics.mask.thickness.hidden (2px composited): click at
   y = screenHeight - 1 over the dock's length -> dock reveals;
   click at y = screenHeight - 6 (outside the 2px band, over a
   visible window) -> the window below receives it.
4. Floating-gap hover reveal: enable a floating dock (screen edge
   margin > 0) with hasFloatingGapInputEventsDisabled satisfied
   (floatingInternalGapIsForced off); verify clicks in the floating
   gap fall through to the desktop (subtractedScreenEdge at work)
   while hovering the dock body still works; with a maximized window
   (hideThickScreenGap), verify the gap closes and input follows.
5. Sidebar block-all: set SidebarOnDemand, toggle it hidden; clicks
   anywhere over the former dock area must fall through (the
   AcceptNoInput -> (-1,-1,1,1) mapping).
6. Parabolic animated band: with parabolic zoom on, glide along the
   tasks; while zoomed (needBothAxis > 0) clicks at the zoomed item's
   TOP pixels (above totals.thickness but inside zoomedForItems) must
   reach the dock, not fall through - the 11f42978 workaround's
   observable behavior. Also confirm no hover jumpiness at the zoomed
   item's outer edge (the original defect).
7. dumpwins geometry cross-check: with debug.graphicsEnabled
   (--with-window flag or config), compare the green mask rectangle
   and dock behavior against dumpwins output for at least bottom+left
   edges, floating and non-floating.
