## Notes / decisions

- Qt5-fidelity arbitration: the inclusive `r<=corner.height()` bound
  in Qt5 is an out-of-bounds read, not behavior to preserve
  (QImage::scanLine valid rows are 0..height()-1); the exclusive
  bound is adopted, as capt did.
- updateRoundnessFallback stays in the adapter: capt's extraction
  scope (15a317ff names four loops) and the spec's responsibility
  line (max opacity, mask roundness, shadow band metrics) both
  exclude it; its column loops are in-bounds.
